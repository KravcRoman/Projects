## parse_mmk.py
Вся необходимая инфа по продуктам есть при листании страницы каталога, поэтому парсинг происходит по страницам каталогов, а не по страницам продуктов.

Парсинг происходит следующим образом:
1. Парсятся ссылки на категории товаров `category_urls = collect_category_urls()`. При помощи requests и bs4.
2. Каждая категория парсится постранично. В функции: parse_products_on_page()
3. После парсинга всего каталога происходит дозапись новой информации в файл `result_mmk.json`
4. И так по циклу для каждой категории.


### Хранение данных
Спарсенные данные хранятся в json файле в формате: `[{}, {}, {}....]`
Где каждый элемент списка (словарь) это информация по отношению продукт-склад. (например, если товар предоставлен в 5 разных складах, то он будет записан в 5 раз).
